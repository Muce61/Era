# 个人量化防过拟合规范

## 1. 文档目标

本规范用于约束个人虚拟币量化研发，防止策略在历史回测区间上被反复调参、被 AI 模型过度优化，最终形成只对历史样本友好的“回测怪物”。

核心原则：

> 策略开发的目标不是让单次回测更好看，而是降低未来实盘失效概率。

---

## 2. 数据集分层

任何策略在开发前，必须明确数据分层。

| 数据集 | 用途 | 是否允许调参 | 规则 |
|---|---|---:|---|
| research_set | 观察市场、生成假设、初步研发 | 是 | 可以看，可以分析，可以试错 |
| validation_set | 验证策略是否有基本泛化能力 | 有限 | 不允许反复调到满意 |
| sealed_test_set | 封存测试集 | 否 | 策略冻结后只能跑一次 |
| forward_test | dry-run / 模拟盘 / 小资金实盘 | 否 | 用真实未来行情验证 |

强制规则：

1. sealed_test_set 一旦被查看，就不能再用于调参。
2. 如果策略在 sealed_test_set 上失败，不允许回头针对 sealed_test_set 修改同一策略。
3. forward_test 失败后，只能进入新策略版本，不允许覆盖旧结果。

---

## 3. 策略研发顺序

禁止直接从代码或参数开始。

正确顺序：

```text
市场假设
↓
策略说明
↓
固定参数范围
↓
研究集回测
↓
验证集测试
↓
封存测试集一次性验证
↓
压力测试
↓
dry-run
↓
小资金实盘
```

错误顺序：

```text
看回测结果
↓
找亏损交易共同点
↓
加过滤条件
↓
收益变好
↓
继续调参
```

---

## 4. 参数规范

参数必须粗粒度，禁止精细雕刻历史。

允许：

```text
EMA = 20 / 50 / 200
ATR_MULTIPLIER = 1.5 / 2.0 / 2.5
RSI_THRESHOLD = 60 / 65 / 70
RISK_PER_TRADE = 0.25% / 0.5% / 1.0%
```

禁止：

```text
EMA = 17
ATR_MULTIPLIER = 1.73
RSI_THRESHOLD = 67.3
VOLUME_RATIO = 3.87
STOP_LOSS = 1.37%
```

判断标准：

> 如果一个参数没有市场含义，只是为了让历史回测好看，它就是过拟合参数。

---

## 5. 禁止行为

以下行为默认视为过拟合风险：

1. 根据某几笔亏损交易直接新增过滤条件。
2. 根据某个币种亏损直接加入黑名单，而不抽象成可解释特征。
3. 根据完整历史区间反复调整 RSI / ADX / 成交量阈值。
4. 用样本外结果反向修改策略。
5. 删除失败策略，只保留盈利策略。
6. 只展示最终最佳回测，不记录尝试次数。
7. 用未来涨幅、未来成交量、未来排名参与当前信号。
8. 为了提高收益曲线而增加无市场假设的规则。

## 6. 策略变更记录

每次策略修改必须记录 change_id。

模板：

```md

策略ID：
修改时间：
修改人：
修改原因：

是否看过 validation_set：是/否

修改后结果：
是否保留：是/否
废弃原因：
```

## 7. 策略失效规则

策略必须提前定义失效条件。

示例：

```text
validation_set Profit Factor < 1.1：废弃
sealed_test_set 最大回撤超过 research_set 的 1.5 倍：废弃
dry-run 连续 3 周无正期望：暂停
实盘连续亏损 3 次：暂停交易
实盘单日亏损达到 2%：触发 kill switch
```

禁止在结果不理想后临时修改失效条件。

---

## 8. 交易指纹库规范

允许记录交易指纹，但禁止“亏什么就过滤什么”。

正确流程：

```text
先定义指纹字段
↓
在所有交易中统一记录
↓
统计不同指纹组合的长期表现
↓
只保留跨区间稳定有效的指纹
```

推荐字段：

```text
symbol
signal_time
fill_time
timeframe
trend_regime
volatility_regime
volume_ratio
rsi
adx
atr_pct
upper_wick_ratio
entry_distance_from_high
mae
mfe
exit_reason
pnl
```

---

## 9. 最低通过标准

一个策略进入 dry-run 前，至少满足：

- [ ] 有 strategy_spec.md
- [ ] 有明确市场假设
- [ ] 参数是粗粒度
- [ ] research_set 通过
- [ ] validation_set 通过
- [ ] sealed_test_set 只运行一次
- [ ] 已做手续费/滑点/延迟压力测试
- [ ] 已做 lookahead 检查
- [ ] 已记录所有失败版本

---

## 10. 核心结论

个人开发者最容易死在“调参成瘾”。

本规范的目的不是提高单次回测收益，而是让每一次策略研发都留下证据链：

```text
为什么做？
怎么验证？
试过多少次？
在哪里失败？
是否真的能泛化？
```
