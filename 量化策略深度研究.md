可以研究，而且你的原交易方式本身就适合改造成一套**多周期事件驱动量化策略**：

> 日线、小时线决定交易方向；15分钟确定所处位置；5分钟识别蜡烛图信号；1分钟负责确认和执行。

但先把核心问题讲清楚：

**蜡烛图形态不是完整策略，只是一个“市场行为特征”。**

《日本蜡烛图技术》后半部分专门讨论蜡烛图与趋势线、均线、摆动指标、成交量的共同验证。书中也明确强调，形态必须结合此前趋势、价格位置及后续确认，而且蜡烛图本身不提供止盈目标。

因此量化目标不是：

```text
发现锤子线 → 立即做多
```

而应该是：

```text
上涨市场
+ 回调至支撑位置
+ 出现看涨蜡烛图
+ 低周期重新转强
→ 做多
```

# 一、把你原来的交易方法拆成五层

## 1. 日线：市场大方向

日线不负责精确进场，只决定允许做什么方向。

初始规则可以定义为：

```text
日线多头：
Close_D > EMA20_D
EMA20_D > EMA60_D
EMA20_D 最近5根斜率 > 0

日线空头：
Close_D < EMA20_D
EMA20_D < EMA60_D
EMA20_D 最近5根斜率 < 0
```

如果日线多头，则：

* 优先寻找做多机会；
* 空头反转形态只作为平多警告；
* 不轻易开逆势空单。

这比看到一个5分钟流星线就直接开空更稳定。

---

## 2. 一小时：当前可交易状态

日线可能上涨，但一小时可能正在急跌，所以还需要一小时过滤。

做多条件：

```text
EMA20_H1 > EMA50_H1
Close_H1 > EMA50_H1
EMA50_H1 斜率 >= 0
```

做空条件反向。

一小时的作用不是寻找蜡烛图，而是判断：

* 顺势上涨；
* 顺势回调；
* 横盘；
* 趋势正在破坏。

第一版策略只交易：

```text
日线与一小时方向一致
```

暂时不做顶底猜测。

---

## 3. 十五分钟：寻找交易位置

蜡烛图形态必须发生在有意义的位置。

做多位置可定义为满足至少一个条件：

```text
价格距离 EMA20_15m 不超过 0.35 × ATR15
价格距离 EMA50_15m 不超过 0.35 × ATR15
价格接近最近20根K线形成的前低或突破位
价格回撤至前一上涨波段的 38.2%～61.8%
```

同时要求：

```text
最近一次回调持续2～10根15分钟K线
回调过程中没有有效跌破一小时趋势结构
```

这里的“接近”不能凭眼睛判断，必须转换成ATR距离：

```text
distance = abs(price - support) / ATR
```

例如：

```text
distance <= 0.35
```

才算接近支撑。

# 二、第一阶段只量化两个形态

不建议一开始把书中几十种形态全部写进去。

那会导致：

* 规则互相重叠；
* 参数数量爆炸；
* 多重检验；
* 回测过拟合；
* 最后无法判断究竟哪个特征有效。

第一版只研究：

1. 锤子线／上吊线；
2. 看涨／看跌吞没形态。

它们形态简单、出现频率较高，也最容易客观化。

---

# 三、K线基础特征

对于任意一根K线：

```python
body = abs(close - open)
range_ = high - low

upper_shadow = high - max(open, close)
lower_shadow = min(open, close) - low

body_ratio = body / range_
upper_ratio = upper_shadow / range_
lower_ratio = lower_shadow / range_
```

需要处理：

```text
range = 0
极小实体
异常插针
交易所坏数据
```

所有形态都应当基于**已经收盘的K线**判断，不能用尚未结束的5分钟K线。

---

# 四、锤子线的量化定义

书中的经典锤子线要求：

* 小实体位于价格区间顶部；
* 下影线至少约为实体的两倍；
* 上影线很短或没有；
* 出现在下降趋势或回调之后。

相同形状出现在上升趋势末端则是上吊线，而且上吊线通常还需要下一时段确认。

## 看涨锤子线

```text
range > 0.5 × ATR5
body_ratio <= 0.35
lower_shadow >= 2 × body
lower_ratio >= 0.55
upper_ratio <= 0.15
close 位于整根K线顶部35%区域
```

位置条件：

```text
发生在15分钟支撑区域
此前5分钟存在至少3根回调K线
或者过去6根5分钟K线累计收益率 < 0
```

代码逻辑：

```python
is_hammer = (
    candle_range >= 0.5 * atr
    and body_ratio <= 0.35
    and lower_shadow >= 2.0 * max(body, min_tick)
    and lower_ratio >= 0.55
    and upper_ratio <= 0.15
    and close >= low + 0.65 * candle_range
)
```

锤子线颜色不是决定性条件，但可以把阳线锤子和阴线锤子分组回测。

# 五、吞没形态的量化定义

书中定义的看涨吞没形态发生在下降趋势中：

* 第一根是较小的阴线实体；
* 第二根是较大的阳线实体；
* 第二根实体覆盖第一根实体。

看跌吞没形态方向相反。

## 看涨吞没

```text
上一根：close[1] < open[1]
当前根：close[0] > open[0]

当前实体下沿 <= 上一根实体下沿
当前实体上沿 >= 上一根实体上沿

当前实体 >= 1.1 × 上一根实体
当前实体 >= 0.5 × ATR5
```

即：

```python
bullish_engulfing = (
    prev_close < prev_open
    and close > open
    and min(open, close) <= min(prev_open, prev_close)
    and max(open, close) >= max(prev_open, prev_close)
    and body >= 1.1 * prev_body
    and body >= 0.5 * atr
)
```

## 加密货币需要特殊处理

虚拟币全天连续交易，通常：

```text
当前K线开盘价 ≈ 上一根K线收盘价
```

因此传统市场中“跳空开盘”的信息在加密市场里基本不存在。

所以不能机械照搬股票市场定义，应重点研究：

* 实体是否真正覆盖；
* 第二根实体是否显著放大；
* 是否发生在支撑阻力；
* 是否伴随成交量放大；
* 后续是否得到确认。

# 六、1分钟进场逻辑

你的“1分钟进场”是合理的，但1分钟只适合做执行，不适合独立决定方向。

在5分钟锤子线或看涨吞没形态收盘后，开启一个最多5分钟的观察窗口。

## 做多触发

满足下面条件：

```text
1分钟收盘价突破5分钟信号K线最高价
或者
1分钟突破最近3根1分钟K线高点
```

并要求：

```text
1分钟收盘价位于EMA20_1m上方
突破K线实体占比 > 50%
```

示例：

```python
entry_trigger = (
    close_1m > signal_high_5m
    and close_1m > ema20_1m
    and body_ratio_1m >= 0.5
)
```

如果5分钟信号出现后，连续5根1分钟K线都没有触发：

```text
信号作废
```

这样可以避免锤子线出现后，价格继续横盘甚至继续下跌，却仍然被迫进场。

倒锤子线和上吊线尤其需要后续确认；书中也将收盘价突破实体视为更有力的确认方式。

```text
条件1：日线 EMA20 > EMA60，日线收盘价 > EMA20
条件2：一小时 EMA20 > EMA50，收盘价 > EMA50
```

# 八、止损、止盈和10倍杠杆

## 止损

做多：

```text
stop = 5分钟形态最低价 - 0.15 × ATR5
```

做空：

```text
stop = 5分钟形态最高价 + 0.15 × ATR5
```

不能固定使用“跌1%止损”，因为不同币种、不同阶段波动率差异很大。

## 第一轮研究的止盈

第一轮回测建议只用：

```text
固定2R止盈
固定1R止损
最长持仓12根5分钟K线
```

原因是先验证**入场信号本身是否有优势**。

如果一开始同时加入：

* 分批止盈；
* 移动止损；
* 保本；
* EMA出场；
* 反向形态出场；

最后即使盈利，也不知道利润来自蜡烛图，还是来自复杂退出规则。

## 仓位公式

设：

```text
账户资金 = equity
单笔风险 = 0.3%
入场到止损距离 = stop_pct
```

则：

```text
名义仓位 = equity × 0.003 / stop_pct
```

例如止损距离为0.6%：

```text
名义仓位 = equity × 0.003 / 0.006
         = equity × 0.5
```

实际只需要0.5倍名义仓位，没必要因为账户允许10倍就强行开10倍。

杠杆应该是仓位计算的结果，而不是输入条件：

```text
最终名义仓位 = min(风险仓位, equity × 10)
```

固定10倍满仓会让策略结果主要由爆仓概率决定，而不是由信号质量决定。

# 九、回测必须设置三组对照实验

这是研究蜡烛图是否真正有效的关键。

## 基准组A：没有蜡烛图

```text
大周期趋势
+ 15分钟回调到支撑
→ 直接进场
```

## 实验组B：加入5分钟蜡烛图

```text
大周期趋势
+ 支撑位置
+ 锤子线／吞没
→ 进场
```

## 实验组C：加入1分钟确认

```text
大周期趋势
+ 支撑位置
+ 5分钟蜡烛图
+ 1分钟突破
→ 进场
```

最后比较：

```text
A vs B：蜡烛图有没有增加有效信息
B vs C：1分钟确认有没有降低假信号
```

如果B不优于A，就说明在该币种和时间周期上，蜡烛图形状没有增量价值。

这比单独看策略最终赚没赚钱重要得多。

# 十、回测实现中的关键要求

建议直接下载1分钟原始数据，再自行聚合：

```text
1m → 5m → 15m → 1h → 1d
```

所有周期必须统一使用同一个交易所和同一个时区，例如UTC。

同时必须处理：

* 手续费；
* 滑点；
* 资金费率；
* 盘口流动性；
* 下单延迟；
* 未收盘K线；
* 多周期数据对齐；
* 同一信号重复入场；
* 同一个币种并发仓位；
* 爆仓与保证金约束。

尤其要避免多周期回测中的未来函数：

```text
在10:31时，不能使用10:00—10:59这根一小时K线的最终收盘数据。
```

# 十一、研究顺序

第一阶段：

```text
趋势回调
+ 锤子线／吞没
+ 1分钟突破
```

第二阶段加入：

```text
流星线
倒锤子线
启明星
黄昏星
十字线
```

启明星、黄昏星在传统定义里含有跳空，但书中也指出，部分实体之间没有理想跳空，并不必然消除形态效力。对于24小时连续交易的虚拟币，应把“严格跳空版”和“无跳空结构版”作为两个不同特征分别回测，不能混为一个信号。

第三阶段再研究：

```text
成交量确认
OI变化
资金费率
主动买卖成交量
波动率状态
趋势与震荡市场分类
```

最终形成的不是一本书里几十个形态的简单集合，而应该是：

```text
市场状态
→ 多周期方向
→ 关键位置
→ 蜡烛图事件
→ 低周期确认
→ 风险和仓位
→ 退出
```

这才是适合虚拟币的蜡烛图量化系统。第一版应默认以 **Binance USDT永续、1分钟原始数据、Python** 实现，先在BTC和ETH上验证增量价值，再扩展到其他币种。
